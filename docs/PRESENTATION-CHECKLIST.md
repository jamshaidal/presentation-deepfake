# Presentation Checklist

## What To Show For Maximum Marks

### Prototype
- Run `npm run model`
- Run `npm run dev`
- Keep 3 sample images ready:
  - one real image
  - one fake image
  - one difficult example
- Keep a backup screen recording in case the live demo fails

### Results To Put On Website
- Accuracy
- Precision
- Recall
- F1-score
- Confusion matrix
- Cross-dataset performance summary

### SRS / Report Material
- Functional requirements
- Interface requirements
- Use case descriptions
- Use case diagram
- Non-functional requirements

### Design Document Material
- Class diagram
- Sequence diagram
- Architectural diagram
- Data flow diagram
- State transition diagram
- ERD
- DB schema

### Viva / Defense Tips
- Start with the problem, not the code
- Explain why EfficientNet-B0 was selected
- Show a live real/fake prediction
- Mention limitations honestly
- End with conclusion and future work
